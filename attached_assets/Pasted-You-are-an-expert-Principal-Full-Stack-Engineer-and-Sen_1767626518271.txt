**CORE FLOWS TO IMPLEMENT (In Order):**

### 1. Public Landing Page (Apple Style)
- **Header:** Minimal, sticky. Logo "ADAPT". Buttons: "Demo", "Login".
- **Hero:** Big H1: "The AI Mentor for Onboarding". Subtext: "No boring PDFs. Train with voice, drills, and feedback."
- **Visuals:** Clean cards, lots of "air" between sections.
- **CTA:** "Try Demo" (Leads to Login).

### 2. Authentication & Roles
- **Login/Register:** Simple form.
- **Role Logic:**
    - **Curator:** Redirect to `/curator/dashboard`.
    - **Employee:** Redirect to `/app/join`.

### 3. Curator Flow: The "2-Minute Magic"
- **Create Track:**
    - Input: Title + Large Text Area ("Paste your internal docs here").
    - Action: "Generate Training".
    - **Backend Logic (Crucial):** The backend calls the AI (or Mock). The AI must convert the text into a JSON array of steps:
        - `type: 'content'` (Short theory, max 3 sentences).
        - `type: 'quiz'` (Multiple choice).
        - `type: 'roleplay'` (Open question: "A client is angry about X. What do you say?").
- **Share:** Display a large 6-digit `join_code`.

### 4. Employee Flow: The "Immersive Player" (The Main Feature)
- **Join:** Enter code -> Load Track.
- **Player UI:** Full screen, no distractions. One card centered.
- **Step Types:**
    - **Content:** Text + "Speaker" icon (Click to read aloud via TTS). Button: "Got it".
    - **Quiz:** Select option. Immediate Green/Red feedback.
    - **Roleplay (Voice):**
        - UI: Question text + Large Pulsing Microphone Button.
        - Interaction: User holds/clicks Mic -> Speaks -> Text appears (STT).
        - Feedback: AI grades the answer (0-10).
- **DRILL MODE LOGIC (Must Implement):**
    - If user answers **WRONG** (Quiz or Roleplay score < 6/10):
        1.  Do NOT advance to the next step.
        2.  Show an "Assistant" avatar/icon.
        3.  Show a **Hint** derived from the knowledge base.
        4.  Reset the input and force the user to try again immediately.
        5.  Log the failure as a "Risk" in the DB.

### 5. Curator Analytics
- **Dashboard:**
    - List of employees.
    - Status: "On Track" (Green) or "Needs Help" (Red - if they triggered Drill Mode > 3 times).
    - Button: "Nudge" (Mock action).
